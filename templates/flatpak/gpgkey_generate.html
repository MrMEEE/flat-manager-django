About GPG Key Generation

The application will use GPG to generate a new unencrypted key pair:

  • Public key for verification
  • Private key for signing (kept secure)
  • Unique fingerprint and key ID

Security Note: Keys are generated without a passphrase so the application can use them for automated signing. Private keys are stored in the database and should never be exposed.
